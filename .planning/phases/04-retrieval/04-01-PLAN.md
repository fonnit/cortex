From prisma/schema.prisma:

```prisma
model Item {
  id        String  @id @default(cuid())
  user_id   String
  status    String  // 'ignored' | 'uncertain' | 'certain' | 'resolved' | 'filed'
  filename  String?
  mime_type String?
  source_metadata Json?  // email headers incl. subject, snippet
  axis_type    String?
  axis_from    String?
  axis_context String?
  proposed_drive_path   String?
  confirmed_drive_path  String?
  embedding  Unsupported("halfvec(512)")?
  ingested_at DateTime @default(now())
}
```

The `embedding` column is typed as `Unsupported("halfvec(512)")`.
Prisma cannot generate or read this column natively — use raw SQL for both write (UPDATE) and index creation.

Stack:
- openai@6.34.0 — `client.embeddings.create({ model: 'text-embedding-3-small', input: [...], dimensions: 512 })`
- @neondatabase/serverless@1.1.0 — `neon(DATABASE_URL)` for raw SQL
- langfuse@3.38.20 — wrap each embed batch in a Langfuse span

Langfuse pattern already established in prior phases:
```typescript

const lf = new Langfuse()
const trace = lf.trace({ name: 'embed', userId })
const span = trace.span({ name: 'openai-embed', input: { count: texts.length } })
// ... call openai ...
span.end({ output: { count: vectors.length } })
await lf.flushAsync()
```

Create `app/api/cron/embed/route.ts` as a Next.js App Router Route Handler.

Cron auth: check `request.headers.get('authorization') === 'Bearer ' + process.env.CRON_SECRET`. Return 401 if mismatch. This matches the pattern from resolve cron.

Logic:
1. Query up to 50 items where `status = 'filed'` AND `embedding IS NULL` using Prisma:
   ```typescript
   const items = await prisma.item.findMany({
     where: { status: 'filed', embedding: null },
     take: 50,
     select: { id: true, filename: true, axis_type: true, axis_from: true, axis_context: true, source_metadata: true, user_id: true },
   })
   ```
   Note: Prisma cannot filter on the Unsupported column — use `embedding: null` as a best-effort; fallback: if Prisma complains, use raw SQL `SELECT id, ... FROM "Item" WHERE status='filed' AND embedding IS NULL LIMIT 50`.

2. Build text for each item using `buildEmbedText(item)` from `lib/embed.ts`.

3. Open a Langfuse trace:
   ```typescript
   const lf = new Langfuse()
   const trace = lf.trace({ name: 'embed-cron', metadata: { count: items.length } })
   const span = trace.span({ name: 'openai-embed', input: { count: items.length } })
   ```

4. Call `embedTexts(texts)` to get vectors array (one vector per item).

5. For each item+vector, write embedding using raw SQL via `@neondatabase/serverless`:
   ```typescript
   import { neon } from '@neondatabase/serverless'
   const sql = neon(process.env.DATABASE_URL!)
   await sql`UPDATE "Item" SET embedding = ${`[${vector.join(',')}]`}::halfvec WHERE id = ${item.id}`
   ```

6. Close Langfuse span: `span.end({ output: { embedded: items.length } })`. Call `await lf.flushAsync()`.

7. Return `Response.json({ embedded: items.length })`.

Error handling: wrap the whole body in try/catch; on error log and return 500.
