<objective>
Convert Stage 2 from a single-shot `claude -p` call into a tool-call loop. The model can call MCP tools mid-reasoning to (a) inspect the path tree, (b) sample what's already filed under any label it's considering, and (c) see where the user has been moving things during triage â€” then commit the same JSON decision shape it returns today.

Purpose: improve Stage 2 placement quality by giving the model on-demand grounding instead of forcing every signal into a single static prompt. Side benefit: the prompt-byte budget shrinks because path-tree dumps move from inline injection to on-demand tool calls.

Output: 2 new HTTP endpoints (Clerk-bypass, requireApiKey-guarded), 1 stdio MCP server in the agent, claude.ts MCP plumbing, restructured Stage 2 prompt, multi-turn-tolerant JSON extraction. The existing /api/classify contract is unchanged.
</objective>
