## Behavioral Diff (CONTEXT decisions)

### D-stage1-routing
**Before:** Every ingested item unconditionally landed in `pending_stage1`.
**After:** A routing decision at ingest time chooses `pending_stage1` only when content is large enough to make Stage 1's relevance gate worthwhile. Per the must-haves:
- A 500 KB Downloads file → `pending_stage2` (skips Stage 1).
- A 2 MiB Downloads file → `pending_stage1` first.
- A Gmail email with no attachments → `pending_stage2`.
- A Gmail email with a >1 MiB attachment → `pending_stage1`.

### D-auto-file
**Before:** Stage 2 always emitted axes; status transitioned to `certain` (all axes ≥0.75) or `uncertain` regardless of confidence ceiling.
**After:** Stage 2 success bodies REQUIRE a `decision` field. When `decision='auto_file'` AND all axes ≥0.85 AND all values are pre-seeded in TaxonomyLabel → status='filed', `confirmed_drive_path` written. Reversibility: triage UI can still override because `axis_*` + `proposed_drive_path` are still populated.

### D-auto-ignore
**Before:** Stage 2 had no terminal-ignore path. Items needed a separate Stage 1 ignore decision OR human triage.
**After:** Stage 2 can return `decision='ignore'` for spam/marketing/installer junk. Confidence ≥0.85 → status='ignored', no axis writes, no Drive resolution. Cold-start guard skipped (no labels committed).

### D-cold-start
**Before:** No vocabulary safety check on auto-actions.
**After:** Auto-file blocked when any axis value isn't in `TaxonomyLabel` for that axis. Auto-ignore explicitly exempted (no labels committed). Trivially satisfied today (closed-vocab prompt) but future-proofs the sibling task that opens up new-label proposals.
