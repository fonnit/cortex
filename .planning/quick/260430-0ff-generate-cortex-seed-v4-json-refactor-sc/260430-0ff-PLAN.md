## v2 → v4 transform rules (from background_for_planning, copy verbatim into transformer)

For each v2 anchor, in order:

1. **Drop date-bucketed paths first.** If `path` matches `^/.+/\d{4}/(\d{2}/)?` → drop the anchor. (Examples that drop: `/travel/2025/...`, `/business/fonnit/invoices-in/2025/...`, `/personal/finance/payslips/2025/...`.)
2. **Type filter.** If `type` is not in `TYPE_V4`:
   - **Special-case remap:** if `type === 'real-estate-permit'` and the file is one of {`TerradanDubai_License.pdf`, `AW REALESTATE 2024 License .pdf`, `PBP Trade License 2026.pdf`}, set `type = 'corporate-registration'` (Decision 5).
   - Otherwise drop the anchor entirely.
3. **From filter.** If `from` is not in `FROM_V4_CORE` and not in `FROM_V4_UTILITIES`, set `from = null` (do not drop).
4. **Terradan path remap** (string replace, in this order on the v2 `path`):
   - `/business/terradan/dubai/` → `/business/terradan-dubai/`
   - `/business/terradan/colombia/` → `/business/terradan-medellin/`
   - `/business/terradan-colombia/` → `/business/terradan-medellin/`
   - For the 3 corporate-registration anchors from step 2: rewrite path to `/business/terradan-dubai/corporate/{filename}`. (v2 had them under `/real-estate/property-deeds/`.)
5. **Drop the `context` field.** Output anchor shape is `{file, type, from, drivePath}` only.

After all anchors processed, build axes:
- `axes.type` = `TYPE_V4` (full list, even if some have zero anchors — taxonomy seed still inserts the labels)
- `axes.from` = `FROM_V4_CORE ∪ (utility providers actually present in surviving anchors)`. Sorted alphabetically. Exclude `null`.

## Zod schema for v4 JSON validation

```typescript
import { z } from 'zod';

const SeedV4Schema = z.object({
  version: z.literal('v4'),
  generated_at: z.string(),
  source: z.object({
    from: z.string(),
    transformer: z.string(),
    decisions: z.string(),
  }).optional(), // optional so hand-edits don't break validation
  axes: z.object({
    type: z.array(z.string()).min(1),
    from: z.array(z.string()).min(1),
  }).strict(), // FAILS if `context` key is present — guards Decision 1
  anchors: z.array(z.object({
    file: z.string().startsWith('/'),
    type: z.string(),
    from: z.string().nullable(),
    drivePath: z.string().startsWith('/'),
  }).strict()).min(1), // FAILS if anchor has `context` field
});
```

Use `.strict()` on both `axes` and the anchor object — that turns "no context allowed" from a soft rule into a load-time failure.
