<objective>
Replace the fixed `/<type>/<from>/<context>/<filename>` Stage 2 path template with a model-defined evolving folder tree. Claude sees the existing confirmed parent directories (with file counts) and proposes a `proposed_drive_path` of arbitrary depth — reusing existing folders when possible, branching when not. Auto-file gates on `path_confidence >= 0.85` AND `parent has ≥3 confirmed items`, replacing u47-3's `allLabelsExist` rule.

Purpose: Stop generating ~1 sibling folder per item under `/invoice/`. The folder tree becomes the user's actual filing history; auto-file only fires once a folder is "stable" (3+ items confirmed there).

Output: New `/api/paths/internal` endpoint, updated Stage 2 prompt + schema, replaced cold-start guard in `/api/classify`, and updated tests on both sides. No Prisma migration. Compatible with the existing axis pipeline (axes still emitted as searchable metadata).
</objective>
